Guía de deploy

Deploy Automático con
GitHub + Docker + VPS

Configura un pipeline de CI/CD completo para que cada push a main despliegue tu aplicación automáticamente.

01
Generar clave SSH en el VPS
SSH

Conéctate a tu VPS y genera un par de claves ED25519. Este algoritmo es más seguro y rápido que RSA.

bash — VPS 
ssh-keygen -t ed25519 -C "tu_email@gmail.com"

Presiona ENTER en todas las preguntas para usar la ruta y passphrase por defecto.

02
Copiar la clave pública
SSH

Muestra el contenido de la clave pública. Cópiala completa — la necesitarás en los pasos siguientes.

bash — VPS 
cat ~/.ssh/id_ed25519.pub

El output empieza con ssh-ed25519 AAAA... — guárdalo en un lugar temporal.

03
Agregar clave a GitHub
Git

Registra tu clave pública en GitHub para poder clonar repositorios via SSH sin contraseña.

  • Ve a github.com → Settings
  • Sección SSH and GPG keys
  • Click en New SSH key
  • Pega el contenido de la clave pública y guarda
04
Clonar repositorio via SSH
Git

En tu VPS, clona el repositorio usando la URL SSH (no HTTPS) para que los pulls futuros sean sin contraseña.

bash — VPS 
git clone git@github.com:usuario/repositorio.git
05
Crear docker-compose.yml
Docker

Define los servicios de tu aplicación. Ajusta el mapeo de puertos según tu stack.

yaml — docker-compose.yml 
version: '3'
services:
  app:
    build: .
    ports:
      - "80:80"
    restart: unless-stopped
06
Crear GitHub Actions workflow
CI/CD

Crea el archivo .github/workflows/deploy.yml en tu repositorio. Se ejecutará en cada push a main.

yaml — .github/workflows/deploy.yml 
name: Deploy
on:
  push:
    branches: [ "main" ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    environment: Despliege_mdm
    steps:
      - name: Deploy via SSH
        uses: appleboy/ssh-action@v0.1.10
        with:
          host: ${{ secrets.SERVER_IP }}
          username: ${{ secrets.SERVER_USER }}
          key: ${{ secrets.SSH_PRIVATE_KEY }}
          script: |
            cd /home/usuario/proyecto
            git pull origin main
            docker compose down
            docker compose up -d --build
07
Configurar Secrets en GitHub
Config

Ve a tu repo → Settings → Secrets and variables → Actions y agrega estas tres variables:

  • SERVER_IP — Dirección IP pública de tu VPS
  • SERVER_USER — Usuario SSH del VPS (ej. root)
  • SSH_PRIVATE_KEY — Contenido completo de ~/.ssh/id_ed25519 (clave privada)
08
Autorizar acceso SSH en el VPS
SSH

Agrega la clave pública del VPS al archivo de claves autorizadas para que GitHub Actions pueda conectarse.

bash — VPS 
nano ~/.ssh/authorized_keys
# Pegar aquí la clave pública (id_ed25519.pub)
09
Corregir permisos SSH
Config

SSH es estricto con los permisos. Si son incorrectos, ignorará las claves autorizadas silenciosamente.

bash — VPS 
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
🚀

¡Pipeline listo!

Cada push a main ejecutará el workflow automáticamente, se conectará a tu VPS via SSH y desplegará la nueva versión con Docker en segundos.
